## 1. 项目介绍（必背，开场白）

> 我做的是一个全栈电商平台，叫淘拼商城。后端用 Django REST Framework，前端用 Vue 3，数据库用 MySQL，缓存用 Redis，异步任务用 Celery。
>
> 项目包含用户注册登录、商品管理、购物车、下单支付、优惠券、评价、搜索这些功能。
>
> 技术亮点有几个：第一，购物车用 Redis Hash 存储，比 MySQL 快很多；第二，创建订单时用事务加行锁防止超卖；第三，用 Celery 做异步任务，订单超时自动取消；第四，优惠券领取用 F() 表达式防止并发超发。

## 2. 技术问答（必背）

### Q：Redis 在你项目里怎么用的？

> 三个场景：
> 1. **购物车**：用 Hash 结构，key 是 `cart:{用户ID}`，field 是 SKU 的 ID，value 是数量和选中状态。好处是可以单独读写某个商品，不用每次都读整个购物车。
> 2. **验证码**：用 String 加 TTL，`SET key value EX 600`，10分钟自动过期。
> 3. **热销商品缓存**：用 String，5分钟过期，减轻数据库压力。

### Q：怎么防止超卖？

> 创建订单时用 `transaction.atomic()` 包裹整个操作，再用 `select_for_update()` 锁住要扣减的 SKU 行。这样两个请求同时来买最后一件商品时，第一个请求拿到锁，第二个请求等待；第一个扣完库存后释放锁，第二个请求发现库存不够，下单失败。保证不会超卖。

### Q：事务是什么？ACID 是什么？

> 事务保证一组操作要么全部成功，要么全部失败。ACID 是四个特性：
> - **原子性**：要么全做要么全不做，转账不能只扣了A没加B
> - **一致性**：前后数据保持一致，转账前后总金额不变
> - **隔离性**：并发事务互不干扰，你转账时别人看不到中间状态
> - **持久性**：提交后永久保存，断电也不丢

### Q：JWT 的双 Token 机制？

> Access Token 有效期2小时，每次请求放在 Header 里验证身份。Refresh Token 有效期7天，只在 Access Token 过期时用。前端拦截器检测到401时，自动用 Refresh Token 换新的 Access Token。好处是 Access Token 泄露了也只有2小时的风险窗口。

### Q：购物车为什么用 Redis 不用 MySQL？

> 购物车读写频率极高，每次浏览、加购、改数量都要操作。Redis 在内存里操作，微秒级响应，比 MySQL 快100倍。而且购物车是临时数据，丢了用户可以重新加购，不需要强一致性。

### Q：SPU 和 SKU 的区别？

> SPU 是商品的抽象概念，比如"iPhone 15 Pro Max"，存公共信息。SKU 是具体规格，比如"黑色 256GB"，存价格、库存。一个 SPU 可以有多个 SKU。分表的好处是避免数据冗余，10个SKU不用存10遍商品描述。

### Q：软删除是什么？为什么用？

> 不真正删除数据，加一个 `is_deleted` 标记。好处是数据可恢复、保留审计记录、不影响外键关联。项目里商品和订单都用了软删除。

### Q：Celery 的 Worker 和 Beat 区别？

> Worker 是干活的人，从队列取任务执行。Beat 是闹钟，按时间间隔往队列塞任务。两个都要启动，定时任务才能跑。项目里 Beat 每5分钟检查超时订单，每小时检查过期优惠券。

### Q：N+1 查询是什么？怎么解决？

> 查一个列表用了1次SQL，遍历列表每个对象访问外键又用了N次，总共N+1次。解决方法：`select_related` 用 JOIN 一次查出来（适合ForeignKey），`prefetch_related` 用额外查询加Python拼接（适合ManyToMany）。

### Q：F() 表达式是什么？

> 在数据库层面操作字段，不用先读到Python再修改。比如 `update(likes=F('likes')+1)` 直接生成 SQL `SET likes = likes + 1`，是原子操作，避免并发问题。

### Q：缓存穿透、击穿、雪崩的区别？

> - **穿透**：查不存在的数据，缓存永远不命中，每次打到数据库。解决：布隆过滤器或缓存空值
> - **击穿**：热点key过期，大量请求同时打到数据库。解决：加锁或永不过期
> - **雪崩**：大量key同时过期，数据库压力骤增。解决：过期时间加随机值

## 3. 项目细节问答

### Q：你的接口怎么限流？

> 用 DRF 的 ScopedRateThrottle。登录接口限制10次/分钟，注册5次/分钟，验证码3次/分钟。底层用 Redis 存请求计数，超过限制返回429。

### Q：优惠券怎么防超发？

> 用 F() 表达式：`UPDATE coupons SET claimed_count = claimed_count + 1 WHERE claimed_count < total_count`。数据库层面原子操作，不会超发。加上 `unique_together` 约束防重复领取。

### Q：支付回调怎么做幂等？

> 检查支付记录状态，如果已经是 success 直接返回成功，不再更新订单。用事务包裹支付记录和订单状态更新，保证一致性。

### Q：订单号怎么生成的？

> 用 UUID 生成，格式是 `ORD` 加16位随机字符串。不用自增ID是因为：安全性（不可预测）、唯一性（分布式不冲突）、不泄露订单数量。

```python
    updated_at = models.DateTimeField('更新时间', auto_now=True)

    class Meta:
        db_table = 'goods_spu'
        ordering = ['-created_at']

    def __str__(self):
        return self.name

    @property
    def price_range(self):
        skus = self.skus.filter(is_active=True)
        if not skus.exists():
            return '暂无报价'
        prices = skus.values_list('price', flat=True)
        min_p, max_p = min(prices), max(prices)
        return f'¥{min_p}' if min_p == max_p else f'¥{min_p} - ¥{max_p}'
```

```python
from rest_framework import serializers
from .models import User

class UserRegisterSerializer(serializers.ModelSerializer):
    password = serializers.CharField(write_only=True, min_length=8)
    password_confirm = serializers.CharField(write_only=True)

    class Meta:
        model = User
        fields = ['username', 'email', 'password', 'password_confirm']

    def validate_username(self, value):
        if User.objects.filter(username=value).exists():
            raise serializers.ValidationError("用户名已存在")
        return value

    def validate(self, data):
        if data['password'] != data['password_confirm']:
            raise serializers.ValidationError({"password_confirm": "两次密码不一致"})
        return data

    def create(self, validated_data):
        validated_data.pop('password_confirm')
        password = validated_data.pop('password')
        user = User(**validated_data)
        user.set_password(password)
        user.save()
        return user
```

**关键点**：
- `write_only=True` 密码不返回
- `validate_字段名` 字段级校验
- `validate` 整体校验
- `set_password()` 加密密码
- `pop()` 取出不需要存库的字段

### 3.2 创建订单（事务 + 行锁）

```python
from django.db import transaction

class OrderCreateView(APIView):
    permission_classes = [IsAuthenticated]

    def post(self, request):
        ser = CreateOrderSerializer(data=request.data)
        ser.is_valid(raise_exception=True)
        items_data = ser.validated_data['items']

        with transaction.atomic():
            total = 0
            order_items = []
            for item in items_data:
                sku = GoodsSKU.objects.select_for_update().get(
                    id=item['sku_id'], is_active=True
                )
                if sku.stock < item['quantity']:
                    return Response({'code': 400, 'message': '库存不足'}, status=400)
                subtotal = sku.price * item['quantity']
                total += subtotal
                order_items.append({'sku': sku, 'quantity': item['quantity'], 'subtotal': subtotal})

            order = Order.objects.create(
                user=request.user,
                order_no=f'ORD{uuid.uuid4().hex[:16].upper()}',
                total_amount=total,
                pay_amount=total,
            )

            for oi in order_items:
                OrderItem.objects.create(
                    order=order, sku_id=oi['sku'].id,
                    price=oi['sku'].price, quantity=oi['quantity'],
                    subtotal=oi['subtotal'],
                )
                oi['sku'].stock -= oi['quantity']
                oi['sku'].save()

        return Response({'code': 200, 'data': OrderSerializer(order).data}, status=201)
```

**关键点**：
- `transaction.atomic()` 事务
- `select_for_update()` 行锁
- 库存检查和扣减在同一个事务里
- UUID 生成订单号

### 3.4 优惠券领取（F() 表达式 + 防重复）

```python
from django.db import transaction, IntegrityError
from django.db.models import F

class CouponClaimView(APIView):
    permission_classes = [IsAuthenticated]

    def post(self, request, pk):
        try:
            coupon = Coupon.objects.get(id=pk, is_active=True)
        except Coupon.DoesNotExist:
            return Response({'code': 404, 'message': '优惠券不存在'}, status=404)

        try:
            with transaction.atomic():
                updated = Coupon.objects.filter(
                    id=pk, claimed_count__lt=F('total_count')
                ).update(claimed_count=F('claimed_count') + 1)
                if not updated:
                    return Response({'code': 400, 'message': '已领完'}, status=400)
                CouponRecord.objects.create(user=request.user, coupon=coupon)
        except IntegrityError:
            return Response({'code': 400, 'message': '已领取过'}, status=400)

        return Response({'code': 200, 'message': '领取成功'})
```

**关键点**：
- `F('claimed_count') + 1` 原子操作
- `claimed_count__lt=F('total_count')` 条件更新
- `IntegrityError` 捕获唯一约束冲突
- 事务包裹

## 5. Celery 任务（必须会写）

### 5.1 定义任务

```python
from celery import shared_task
from django.utils import timezone
from datetime import timedelta

@shared_task(name='apps.orders.tasks.cancel_expired_orders')
def cancel_expired_orders():
    from .models import Order
    from apps.goods.models import GoodsSKU

    deadline = timezone.now() - timedelta(minutes=30)
    expired = Order.objects.filter(status='pending', created_at__lt=deadline)

    count = 0
    for order in expired:
        for item in order.items.all():
            try:
                sku = GoodsSKU.objects.get(id=item.sku_id)
                sku.stock += item.quantity
                sku.sales = max(0, sku.sales - item.quantity)
                sku.save()
            except GoodsSKU.DoesNotExist:
                pass
        order.status = 'cancelled'
        order.save()
        count += 1
    return f'cancelled {count} orders'
```

```sql
-- 1. 找出消费最多的前10个用户
SELECT u.username, SUM(o.pay_amount) AS total
FROM users u
JOIN orders o ON u.id = o.user_id
WHERE o.status = 'completed'
GROUP BY u.id
ORDER BY total DESC
LIMIT 10;

-- 2. 统计每个分类的商品数量
SELECT c.name, COUNT(s.id) AS count
FROM goods_category c
LEFT JOIN goods_spu s ON c.id = s.category_id AND s.is_deleted = 0
GROUP BY c.id;

-- 3. 找出从未下过单的用户
SELECT u.username
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE o.id IS NULL;

-- 4. 查询最近7天每天的订单总额
SELECT DATE(created_at) AS day, SUM(pay_amount) AS total
FROM orders
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY)
GROUP BY DATE(created_at)
ORDER BY day;

-- 5. 找出库存为0但还在架的商品
SELECT s.name, s.id
FROM goods_spu s
JOIN goods_sku k ON s.id = k.spu_id
WHERE s.is_on_sale = 1 AND k.stock = 0;

-- 6. 给商品名字段加索引
CREATE INDEX idx_name ON goods_spu(name);

-- WHERE a=1 ✅

-- WHERE a=1 AND b=2 AND c=3 ✅
```

```python
        print(f'调用 {func.__name__}')
        return func(*args, **kwargs)
    return wrapper

@log
def add(a, b):
    return a + b

# 4. 生成器
def fibonacci():
    a, b = 0, 1
    while True:
        yield a
        a, b = b, a + b

finally:
```
